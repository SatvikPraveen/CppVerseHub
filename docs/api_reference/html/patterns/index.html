Gang of Four and modern design pattern implementations in C++

Singleton Pattern
Creational Pattern
Thread-safe singleton implementation ensuring single instance creation with lazy initialization and proper cleanup.

Implementation Classes

  • ResourceManager
    Global resource coordinator
  • ConfigManager
    Application configuration handler
  • Logger
    System-wide logging service
auto& manager = ResourceManager::getInstance(); manager.allocateResources("minerals", 1000);

Key Benefits

  • Guaranteed single instance
  • Global access point
  • Lazy initialization
  • Thread-safe implementation
Advanced
Builder Pattern
Creational Pattern
Constructs complex entities step by step, providing a fluent interface for creating fleets, planets, and missions with multiple configuration options.

Implementation Classes

  • FleetBuilder
    Fleet construction director
  • PlanetBuilder
    Planet configuration builder
  • MissionBuilder
    Mission parameter builder
  • EntityDirector
    Construction orchestrator
auto fleet = FleetBuilder() .setName("Alpha Squadron") .addShip("Destroyer", 3) .addShip("Frigate", 5) .setHomeBase(planet) .build();

Key Benefits

  • Step-by-step object construction
  • Fluent interface design
  • Complex parameter validation
  • Immutable result objects

State Pattern
Behavioral Pattern
Manages mission states and entity lifecycle through state objects, enabling clean state transitions and state-specific behavior.

Implementation Classes

  • MissionState
    Abstract state interface
  • PendingState
    Mission awaiting execution
  • ExecutingState
    Mission in progress
  • CompletedState
    Mission finished
mission.setState(std::make_unique<ExecutingState>()); mission.process(); // Behavior depends on current state

Key Benefits

  • Clean state transitions
  • State-specific behavior encapsulation
  • Eliminates complex conditionals
  • Easy state machine visualization
Advanced
Adapter Pattern
Structural Pattern
Enables integration with legacy systems and third-party libraries by adapting incompatible interfaces to work with the modern codebase.

Implementation Classes

  • LegacySystemAdapter
    Legacy interface adapter
  • DatabaseAdapter
    External database interface
  • NetworkAdapter
    Network protocol adapter
  • FileFormatAdapter
    File format converter
auto adapter = std::make_unique<LegacySystemAdapter>(legacySystem); modernInterface.processData(adapter.get());

Key Benefits

  • Legacy system integration
  • Interface compatibility
  • Code reuse without modification
  • Separation of concerns

Strategy Pattern
Behavioral Pattern
Encapsulates algorithms in separate classes, allowing runtime selection of different behaviors for fleet routing and mission execution.

Implementation Classes

  • RoutingStrategy
    Abstract routing algorithm
  • ShortestPathRouting
    Optimal distance routing
  • SafestPathRouting
    Risk-minimized routing
  • EconomicalRouting
    Resource-efficient routing
fleet.setRoutingStrategy( std::make_unique<ShortestPathRouting>() ); fleet.navigateTo(destination);

Key Benefits

  • Runtime algorithm switching
  • Easy to extend with new strategies
  • Eliminates conditional complexity
  • Open/closed principle compliance
Intermediate
Observer Pattern
Behavioral Pattern
Implements publisher-subscriber mechanism for planet monitoring, resource updates, and event notifications across the system.

Implementation Classes

  • Observable
    Subject interface
  • Observer
    Listener interface
  • PlanetMonitor
    Planet state observer
  • ResourceTracker
    Resource change observer
planet.addObserver(std::make_shared<ResourceTracker>()); planet.updateResources(); // Notifies all observers

Key Benefits

  • Loose coupling between objects
  • Dynamic observer registration
  • Event-driven architecture
  • Broadcast communication
Intermediate
Decorator Pattern
Structural Pattern
Adds new functionality to missions and entities dynamically without altering their structure through composition-based enhancement.

Implementation Classes

  • MissionDecorator
    Abstract decorator base
  • UrgentMissionDecorator
    Priority enhancement
  • StealthMissionDecorator
    Stealth capability
  • ReconMissionDecorator
    Intelligence gathering
auto mission = std::make_unique<ExplorationMission>(); mission = std::make_unique<UrgentMissionDecorator>( std::move(mission) );

Key Benefits

  • Runtime behavior enhancement
  • Flexible combination of features
  • Preserves original object interface
  • Single responsibility principle
Advanced
Command Pattern
Behavioral Pattern
Encapsulates fleet operations as objects, enabling queuing, logging, and undo functionality for complex mission management.

Implementation Classes

  • Command
    Abstract command interface
  • MoveFleetCommand
    Fleet movement operation
  • AttackCommand
    Combat operation
  • CommandInvoker
    Command execution manager
auto cmd = std::make_unique<MoveFleetCommand>(fleet, target); invoker.executeCommand(std::move(cmd)); invoker.undo(); // Reverse last command

Key Benefits

  • Operation queuing and batching
  • Undo/redo functionality
  • Command logging and replay
  • Decoupled sender-receiver
